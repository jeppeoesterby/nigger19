How to give feedback that improves the prompt

Feedback is the raw data that drives prompt iteration. Every mark you make feeds two things: Insights (where the prompt is weak) and regression tests (verifying that prompt changes actually help).

Workflow

  1. Pick a past run (at the bottom of this dashboard).
  2. For each (invoice, model) cell: click Add.
  3. For each error the model flagged, set the verdict — correct, false_positive, or wrong_amount (right line, wrong number).
  4. Add errors the model missed — this is the most valuable signal for prompt tuning. Provide: item number, description, charged price, agreed price, overpayment.
  5. Tag agreement type (price_agreement / index_agreement / project_based / framework / other) and the canonical supplier name (e.g. "STARK").
  6. Save.

You DON'T need to

  • Review every invoice — even 5-10 thorough reviews give usable data.
  • Mark everything as correct — only mark what you're confident about; leave the rest blank.
  • Fix the model — just describe what was wrong, the system will use it to tune the prompt.

Pro tips

  • Missed errors are gold — prioritize them over verdicts on already-found errors.
  • Use wrong_amount when the model identifies the right line but sets a wrong number (often a sign of a subtotal or quantity error).
  • Concrete notes help: "model didn't account for quarterly index adjustment" beats "wrong".
  • Once you've collected feedback on several invoices, run a regression test after each prompt change to verify it's a net improvement.
